## ✨ 새로운 기능

### 1. 분자 비교 (Molecular Comparison)
- **기능**: 두 분자의 Tanimoto 유사도 계산
- **UI**: 새로운 "분자 비교" 버튼 (도구 모음)
- **출력**: 
  - 유사도 점수 (0.0 ~ 1.0)
  - 동일 분자 여부 판별
  - 공통 부분구조 식별
  - 구조적 차이점 분석
- **사용자 입력**:
  - 분자 2 SMILES 또는 분자식
  - 실시간 캔버스 데이터 (분자 1)

### 2. 계산 히스토리 브라우저
- **기능**: ORCA 계산 결과 저장 및 검색
- **UI**: 새로운 "계산 히스토리" 버튼 (도구 모음)
- **테이블 컬럼**:
  - 계산 ID
  - 타임스탬프
  - 분자식
  - 계산 방법 (B3LYP, 등)
  - 에너지 (하트리)
  - 수렴 상태
  - 계산 시간
- **검색 기능**:
  - 분자식, 방법, 날짜로 검색
  - 실시간 필터링
  - 상세 정보 조회

### 3. 배치 처리 (Batch Processing)
- **기능**: 다중 분자 순차 처리
- **UI**: 새로운 "배치 처리" 버튼 (도구 모음)
- **입력**:
  - 다중 SMILES (줄바꿈으로 구분)
  - 계산 방법, 기저 집합 (선택사항)
- **출력**:
  - 실시간 진행률 표시 (0% ~ 100%)
  - 처리 결과 목록
  - 완료 시간 통계

### 4. 올가미 선택 (Lasso Select)
- **기능**: Theory 레이어에서 자유 형태 원자 선택
- **UI**: "올가미 선택" 버튼 (Theory 레이어 우측 하단, 주황색)
- **사용법**:
  - 마우스 좌측 드래그로 경로 그리기
  - 올가미 내부 원자 자동 선택
  - 주황색 실선으로 경로 표시
- **최적화**: 
  - 5픽셀 데시메이션 (부드러운 곡선)
  - 리얼타임 렌더링

## 🐛 알려진 문제 및 제한사항

### 1. SMILES 제약
- 매우 복잡한 구조 (>100 원자) 생성 실패 가능
- 불안정한 고리 (3-고리 등) 자동 정규화

### 2. 배치 처리
- 현재: 단일 CPU 스레드 (순차 처리)
- 계산 시간이 오래 걸리는 경우 응답성 저하 가능

### 3. 히스토리 검색
- 대량 데이터 (>10K 항목) 시 검색 느려짐
- 향후 데이터베이스 인덱싱으로 개선 예정

### 4. 올가미 선택
- Theory 레이어 전용 (다른 레이어에서는 비활성)
- 매우 빠른 드래그 시 일부 포인트 누락 가능
