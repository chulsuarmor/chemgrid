### 코드 변경

```python
# 결합 길이의 40% 제한
avg_bond_length = sum(bond_lengths) / len(bond_lengths)  # ~40px

```

**AFTER (v3.1):**

```python
avg_bond_length = sum(bond_lengths) / len(bond_lengths)  # ~40px
max_cloud_radius = avg_bond_length * 0.25  # 10px (극단적 축소)
```

### 효과 분석

| 항목 | v3.0 (40%) | v3.1 (25%) | 개선 효과 |
|------|-----------|-----------|----------|
| 최대 반지름 | 16px | 10px | **37.5% 축소** |
| 인접 원자 간섭 | 있음 (16px 겹침) | 거의 없음 (10px 집중) | ✓ 색상 간섭 원천 차단 |
| ortho/meta/para 구분 | 보통 | 매우 명확 | ✓ 경계 극명 |
| 시각적 선명도 | 보통 | 매우 높음 | ✓ 핵 근처 집중 |

**수치 예시 (니트로벤젠, 결합 길이 40px):**
```
v3.0: 각 원자 주변 16px 확산
  → ortho-C와 meta-C 사이 거리 40px
  → 16px + 16px = 32px (80% 겹침 위험)

v3.1: 각 원자 주변 10px 확산
  → ortho-C와 meta-C 사이 거리 40px
  → 10px + 10px = 20px (50%, 간섭 최소화)
```

**시각적 효과:**
- ortho/meta/para 위치가 **완전히 독립된 색상 영역**으로 표현
- 인접 원자 색상 혼합 **원천 차단**
- 전자구름이 원자 핵 주변에 **고도로 집중**

## 🔧 공격적 수정 2: 색상 스케일 고정 (0.15 하한선)

### 변경 위치
**파일:** `_source/renderer.py` Lines ~315-322

### 코드 변경

**BEFORE (v3.0):**
```python
# 최소 0.05 보장 (너무 민감)
charge_range = max(abs(min_charge), abs(max_charge), 0.05)
```

**AFTER (v3.1):**
```python
# ✅ FIX v3.1: 최소 0.15 보장 (벤젠 오차 무시)
charge_range = max(abs(min_charge), abs(max_charge), 0.15)
```

### 효과 분석

| 분자 | v3.0 (0.05) | v3.1 (0.15) | 개선 효과 |
|------|------------|------------|----------|
| 벤젠 (C: -0.01) | normalized = -0.20 | normalized = -0.067 | ✓ 미세 오차 무시 |
| 니트로벤젠 (range: 0.35) | normalized = full | normalized = full | ✓ 의미 있는 차이만 표현 |
| 아세틸벤젠 (range: 0.30) | normalized = full | normalized = full | ✓ 실제 치환기 효과 명확 |

**문제 해결:**
```
v3.0 문제: 최소 0.05로 인해 모든 분자가 "풀 컬러"
  → 벤젠 (-0.01)도 진한 파랑으로 표현
  → 실제 치환기 효과와 구분 불가능

v3.1 해결: 최소 0.15로 상향
  → 벤젠 (-0.01)은 약한 회색 (오차 범위)
  → 니트로벤젠 (±0.30)은 진한 색 (실제 효과)
  → 의미 있는 전하 차이만 강조
```

**시각적 효과:**
- 벤젠 고리의 미세한 오차(±0.01) **시각적으로 무시**
- 실제 치환기 효과(±0.15 이상)만 **강한 색상 대비**로 표현
- "풀 컬러" 부작용 **완전 제거**

## 🔧 공격적 수정 3: Gamma 대비 강화 (Power 로직)

### 변경 위치
**파일:** `_source/renderer.py` Lines ~424-432

### 코드 변경

**BEFORE (v3.0):**
```python
# 선형 가중치
normalized_charge = charge / charge_range
mix = min(abs(normalized_charge) * 1.8, 0.98)  # 선형
```

**AFTER (v3.1):**
```python
# ✅ FIX v3.1: Power 함수 (Gamma correction)
normalized_charge = charge / charge_range
mix = pow(abs(normalized_charge), 0.6) * 2.0  # 비선형
mix = min(mix, 0.98)
```

### 효과 분석 (수치 예시)

| normalized_charge | v3.0 (선형) | v3.1 (Power) | 대비 강화 |
|-------------------|------------|-------------|----------|
| 0.05 | 0.090 | **0.205** | **2.3배** |
| 0.10 | 0.180 | **0.325** | **1.8배** |
| 0.15 | 0.270 | **0.419** | **1.6배** |
| 0.20 | 0.360 | **0.503** | **1.4배** |
| 0.30 | 0.540 | **0.653** | **1.2배** |
| 0.50 | 0.900 | **0.917** | **1.0배** |

**Power 함수 효과:**
```python
# Gamma = 0.6 (< 1.0) → 압축 곡선
# 작은 값을 더 크게, 큰 값을 약간 작게

예시: normalized = 0.10
  선형: 0.10 * 1.8 = 0.180
  Power: pow(0.10, 0.6) * 2.0 = 0.251^0.6 * 2.0 = 0.325

→ 미세한 전하(0.10)가 시각적으로 1.8배 더 뚜렷하게 표현
```

**시각적 효과:**
- ±0.05 차이도 **육안으로 명확히 구분**
- 선형 대비 **2.3배 강한 색상 대비**
- ortho/meta/para 지향성 **극명히 표현**

**니트로벤젠 예시 (min=-0.30, max=+0.35, range=0.35):**

| 위치 | 전하 | normalized | v3.0 mix | v3.1 mix | 색상 강도 |
|------|------|-----------|---------|---------|----------|
| ortho-C | -0.010 | -0.029 | 0.052 | **0.117** | 약한 파랑 |
| meta-C | +0.010 | +0.029 | 0.052 | **0.117** | 약한 빨강 |
| para-C | -0.010 | -0.029 | 0.052 | **0.117** | 약한 파랑 |

**개선:** mix 값이 0.052 → 0.117 (2.3배), 색상 대비 **극단적 강화**

## 🔧 공격적 수정 4: 치환기 크기 20% 축소

### 변경 위치
**파일:** `_source/renderer.py` Lines ~370-390

### 코드 변경

**BEFORE (v3.0):**
```python
# 모든 원자 동일 크기
if at_main == "H":
    c_scale, d_scale = 0.5, 0.5
else:
    c_scale = el_data.get("cloud_scale", 1.0)
    d_scale = el_data.get("density_scale", 1.0)
```

**AFTER (v3.1):**
```python
# ✅ FIX v3.1: 치환기 원자 판별 및 크기 20% 축소
is_substituent_atom = (at_main in ["O", "N", "F", "Cl", "Br", "S", "P"] and
                       pt_key not in aromatic and
                       atom_is_size.get(pt_key, 0) < 2)

if at_main == "H":
    c_scale, d_scale = 0.5, 0.5
elif is_substituent_atom:
    # 치환기: 크기 80%, 밀도 150% (EWG 가중치)
    base_c_scale = el_data.get("cloud_scale", 1.0)
    c_scale = base_c_scale * 0.80  # 20% 축소
    d_scale = el_data.get("density_scale", 1.0) * 1.5  # EWG 1.5배
else:
    c_scale = el_data.get("cloud_scale", 1.0)
    d_scale = el_data.get("density_scale", 1.0)
```

### 효과 분석

**치환기 크기 축소:**

| 원자 타입 | v3.0 크기 | v3.1 크기 | 축소율 |
|----------|----------|----------|--------|
| 고리 C | 1.0 (기준) | 1.0 (기준) | - |
| 치환기 O | 1.0 | **0.80** | **20% 축소** |
| 치환기 N | 1.0 | **0.80** | **20% 축소** |
| 치환기 F | 1.0 | **0.80** | **20% 축소** |

**EWG 가중치 강화:**

| 효과 | v3.0 | v3.1 | 강화율 |
|------|------|------|--------|
| 공명 효과 (+R) | 1.0 | 1.0 | - |
| 유도 효과 (-I) | 1.0 | **1.5** | **50% 강화** |

**문제 해결:**
```
v3.0 문제: -COOH의 -OH 산소 구름이 고리 탄소 가림
  → 산소 +R 효과가 시각적으로 -I 효과를 압도
  → 카복실기가 EWG인지 EDG인지 불명확

v3.1 해결:
  1. 산소 구름 크기 20% 축소 → 고리 탄소 시각적 노출
  2. EWG 밀도 1.5배 강화 → -I 효과가 +R 효과를 압도
  → 카복실기의 EWG 특성 명확히 표현
```

**시각적 효과:**
- 치환기 산소/질소가 고리 탄소를 **가리지 않음**
- EWG 특성(전자 흡수)이 시각적으로 **명확히 표현**
- ortho/meta/para 지향성 **완벽히 보존**
