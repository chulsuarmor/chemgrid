## 🎯 v3.2의 핵심 철학

### 문제점 (v3.1의 한계)

```
v3.1 엔진은 물리적으로 정확하지만, 화학적 통찰을 주지 못함:
- ±0.01 전하 차이를 색상 대비로 표현했으나 여전히 구분 어려움
- meta/ortho/para 위치가 시각적으로 동등하게 보임
- 치환기 효과가 고리 탄소 정보를 묻어버림
- 어느 위치가 친전자성 공격을 받을지 직관적으로 알 수 없음
```

### 해결책 (v3.2의 이론적 가공)

```
화학적으로 의미 있는 패턴을 증폭하고, 노이즈를 억제:
1. 반응성 가중치: 평균 대비 전하 편차를 지수 함수로 증폭
2. 가변 구름 크기: 활성화/비활성화 탄소를 크기로 시각적 구분
3. 조준선 마커: 가장 반응성 높은 2-3개 탄소 자동 강조
4. 레이어 강도: 고리 탄소 정보가 치환기에 묻히지 않게 보호
```

## 📐 4가지 이론적 가공 로직

### 1. 반응성 가중치 (Reactivity Weight): 지수 함수 증폭

#### 기존 문제 (v3.1)
```python
# 니트로벤젠 (NO₂-C₆H₅)
ring_avg_charge = (ortho + meta + para + ...) / 6 = -0.003

meta-C:  +0.010  →  deviation = +0.010 - (-0.003) = +0.013
ortho-C: -0.010  →  deviation = -0.010 - (-0.003) = -0.007

# v3.1 선형 스케일링
normalized_meta  = +0.010 / 0.35 = 0.029
normalized_ortho = -0.010 / 0.35 = -0.029

mix_meta  = pow(0.029, 0.6) * 2.0 = 0.329 (32.9% 빨강)
mix_ortho = pow(0.029, 0.6) * 2.0 = 0.329 (32.9% 파랑)

문제: 색상 강도가 동일 (32.9%) → 구분 어려움
```

#### v3.2 해결책: 지수 증폭
```python
# 지수 가중치 계산
charge_deviation_meta  = abs(+0.013)
charge_deviation_ortho = abs(-0.007)

reactivity_weight_meta  = exp(0.013 * 35) = exp(0.455) = 1.576 (57.6% 증폭)
reactivity_weight_ortho = exp(0.007 * 35) = exp(0.245) = 1.278 (27.8% 증폭)

# 색상 강도에 가중치 적용
mix_meta  = 0.329 * 1.576 = 0.519 (51.9% 빨강) ✓
mix_ortho = 0.329 * 1.278 = 0.420 (42.0% 파랑) ✓

효과: 51.9% vs 42.0% = 23.5% 상대 차이 (눈으로 명확히 구분)
```

#### 수학적 튜닝
```python
# 지수 상수 k=35 선택 근거

목표: ±0.01 차이 → 50%+ 시각적 차이

k * Δcharge = 35 * 0.01 = 0.35
exp(0.35) ≈ 1.419 (42% 증폭)

±0.02: exp(35 * 0.02) = exp(0.70) = 2.014 (101% 증폭, 2배)
±0.03: exp(35 * 0.03) = exp(1.05) = 2.858 (186% 증폭, 최대치)

max_weight = 3.0 (안전 상한선)
```

### 2. 가변 구름 크기 (Variable Cloud Size): 활성/비활성 차별화

#### 개념
```
EWG (전자끌개) 치환기:
- 고리 평균보다 양전하(+) 큰 탄소 = 비활성화 (Deactivated)
  → 전자구름이 "위축"되어야 함
  → 시각적으로 작고 약하게 보임

- 고리 평균보다 음전하(-) 큰 탄소 = 활성화 (Activated)
  → 전자구름이 "풍성"해야 함
  → 시각적으로 크고 강하게 보임
```

#### 구현
```python
# 니트로벤젠 예시
ring_avg_charge = -0.003

ortho-C: -0.010 < -0.003  →  활성화 (전자 풍부)
meta-C:  +0.010 > -0.003  →  비활성화 (전자 부족)
para-C:  -0.010 < -0.003  →  활성화 (전자 풍부)

# 기본 반지름 계산
base_radius = (19.5 + log(...) * 15.0 + strength * 7.5) * c_scale
radius = min(base_radius, avg_bond_length * 0.25)  # 10px (v3.1)

# 비활성 탄소 추가 축소
if is_ring_carbon and charge > ring_avg_charge:
    radius *= 0.60  # 40% 추가 축소

# 결과
meta-C (비활성):  10px * 0.60 = 6px  (위축됨) ✓
ortho-C (활성):   10px * 1.00 = 10px (정상) ✓
para-C (활성):    10px * 1.00 = 10px (정상) ✓
```

#### 시각적 효과
```
니트로벤젠을 Lewis 레이어로 보면:

        NO₂
         ↓
    [ortho: 10px 구름] ← 활성, 풍성
    /                  \
[meta: 6px 구름]   [para: 10px 구름] ← meta는 위축, 명확히 구분
    \                  /
    [meta: 6px 구름]
         ↓
    [ortho: 10px 구름]

친전자성 공격 위치를 시각적으로 즉시 파악 가능
```

### 3. 조준선 마커 (Crosshair Markers): 심볼릭 인터페이스

#### R/S 표기 확장
```
기존: 카이랄 중심에 (R) 또는 (S) 빨강 라벨
      → 입체화학 정보 제공

v3.2: 반응성 중심에 녹색 십자선 마커
      → 친전자성 공격 위치 자동 표시
```

#### 알고리즘
```python
# 1단계: 고리 탄소 중 전자 밀도 상위 수집
electron_rich_carbons = []
for carbon in ring_carbons:
    if carbon.charge < ring_avg_charge:  # 평균보다 음전하
        electron_rich_carbons.append((position, charge))

# 2단계: 전하 순으로 정렬 (가장 음인 순)
electron_rich_carbons.sort(key=lambda x: x[1])

# 3단계: 상위 2-3개 선택
num_markers = min(3, max(2, len(electron_rich_carbons) // 3))
top_sites = electron_rich_carbons[:num_markers]

# 4단계: 녹색 십자선 + 원 마커 렌더링
for position, _ in top_sites:
    draw_crosshair(position, color=(0, 200, 0), size=12px)
    draw_circle(position, radius=6px)
```

#### 니트로벤젠 예시
```
전하 정렬:
1. ortho-C: -0.010 (가장 음) ← 마커 1
2. para-C:  -0.010 (공동 2위) ← 마커 2
3. ortho-C: -0.010 (공동 2위) ← 마커 3
4. meta-C:  +0.010 (양전하, 제외)

시각화:
        NO₂
         ↓
    [ortho ⊕] ← 녹색 조준선
    /          \
  meta      [para ⊕] ← 녹색 조준선
    \          /
    meta
         ↓
    [ortho ⊕] ← 녹색 조준선

사용자가 즉시 이해: "ortho/para가 반응성 높음"
```

### 4. 고리 탄소 레이어 강도 (Ring Carbon Layer Strength)

#### 문제점 (v3.1)
```
니트로벤젠 (NO₂-C₆H₅):

치환기 O: -0.300 전하 → 매우 진한 파랑 구름
고리 C:   ±0.010 전하 → 약한 빨강/파랑 구름

렌더링 순서: O (먼저) → 고리 C (나중)
             치환기    고리

결과: 고리 탄소가 위에 그려지지만,
     치환기 O의 색상이 너무 강렬해서
     고리 탄소의 미세한 전하 차이가 묻혀버림
```

#### v3.2 해결책
```python
# 불투명도 계산
base_layer_alpha = base_alpha * min(max(strength, charge_intensity * 1.1), 1.5)

# v3.1: 모든 원자 동일
alpha = int(base_layer_alpha)

# v3.2: 고리 탄소만 1.5배 강화
if is_ring_carbon:
    base_layer_alpha *= 1.5  # 1.5배 불투명도 강화

alpha = int(base_layer_alpha)
```

#### 효과 비교
```
치환기 O (v3.1 = v3.2):
  charge: -0.300
  alpha = 140 * 1.5 = 210

고리 C (v3.1):
  charge: ±0.010
  alpha = 140 * 1.0 = 140

고리 C (v3.2):
  charge: ±0.010
  alpha = 140 * 1.0 * 1.5 = 210  ← O와 동등한 불투명도

결과:
v3.1: O의 진한 파랑이 고리 C의 약한 빨강을 압도
      → 고리 전하 정보 손실

v3.2: 고리 C가 O와 동등한 불투명도로 위에 렌더링
      → 고리 전하 정보 명확히 보존
      → EWG 효과가 묻히지 않음
```

## 📈 성능 영향 분석

### 추가 계산량

```python
# 1. 고리 탄소 평균 계산: O(n)
ring_avg_charge = sum(ring_carbons) / len(ring_carbons)

# 2. 반응성 가중치: 각 고리 탄소마다 O(1)
reactivity_weight = exp(abs(charge - ring_avg) * 35.0)

# 3. 조준선 마커 정렬: O(n log n)
electron_rich_carbons.sort(key=lambda x: x[1])

# 4. 마커 렌더링: 최대 3개 = O(1)
```

### 실측 오버헤드 (예상)

| 분자 크기 | v3.1 렌더링 | v3.2 렌더링 | 오버헤드 |
|---------|-----------|-----------|----------|
| 소형 (~20 atoms) | < 2ms | < 3ms | +50% |
| 중형 (~50 atoms) | < 5ms | < 7ms | +40% |
| 대형 (~180 atoms) | < 12ms | < 16ms | +33% |
