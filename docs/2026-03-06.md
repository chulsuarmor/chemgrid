## 3. 화학 엔진 문제점 (다음 세션 수정 대상)

### [CHEM-1] 공명구조 비편재화 계산 오류
- **증상**: 벤젠, 나프탈렌, 피롤 등 방향족 분자에서 전자구름이 균등 비편재화되지 않음
- **올바른 방법**: Hückel 이론 또는 RDKit의 `Chem.GetAromaticAtoms()` 기반으로 방향족 원자에 전하 0.5씩 분배
- **현황**: `engine_core.py`의 공명 기여도 계산이 단순 규칙 기반으로 실제 비편재화를 표현 못함

### [CHEM-2] 양전하/음전하 분자 전자구름 오류
- **증상**: NH4+, OH-, NO2-, SO4^2- 등 이온 분자에서 전하가 반영된 전자밀도가 잘못됨
- **올바른 방법**: `atoms["charge"]` 필드를 analyzer에서 읽어 Mulliken 전하 계산에 반영
- **현황**: `analyzer.py`의 `analyze()` 메서드가 `charge` 필드를 전자구름 계산에 미반영

### [CHEM-3] 라디칼 분자 전자구름 오류
- **증상**: CH3·, NO· 등 라디칼 분자에서 홀전자 표현이 전자구름에 미반영
- **올바른 방법**: `atoms["attach"]`의 `"·"` (Radical 기호)를 단전자 밀도로 변환
- **현황**: 라디칼이 시각적으로만 표시되고 전자구름 계산에 영향 없음

### [CHEM-4] ORCA DFT 인터페이스 — ESP 전자구름 연동 미완
- **위치**: `canvas.py`의 `on_orca_calculation_complete()` → `ElectronDensityAnalyzer` 임포트 실패 (모듈 없음)
- **올바른 방법**: ORCA .out 파일의 Mulliken 전하를 직접 읽어 각 원자의 `effective_charge` 설정

## 4. 중장기 구축 계획 (사용자 지시 2026-03-06)

### [계획-A] 관용명 자동 그리기 시스템 (최우선 신규 기능)
- **목표**: 메인 그리기 레이어 하단 텍스트 입력 바에 관용명(예: "benzene", "아스피린")을 입력하면 그리기 도구가 자동으로 분자를 그려주는 시스템
- **구현 방식**: 
  1. PubChem API: 관용명 → SMILES 변환 (`GET /compound/name/{name}/property/IsomericSMILES/JSON`)
  2. SMILES → 원자/결합 딕셔너리 변환 (RDKit `MolFromSmiles` + 2D 좌표 생성)
  3. 이미지 인식 피드백 그리기 마우스 경로형 메커니즘으로 캔버스에 그리기
- **평가**: **구현 가능** (난이도 중간)
  - 분당 호출 제한 있는 Google AI Studio/Gemini로도 가능: SMILES 생성은 PubChem으로, 좌표 계산은 RDKit으로 처리하면 API 호출 최소화
  - 현재 `agents/02_canvas_interaction/text_input_handler.py` 파일이 이미 존재 → 이 파일 확장
- **구현 위치**: `agents/02_canvas_interaction/text_input_handler.py`
- **작업 단계**:
  1. 텍스트 입력 바 UI (`QLineEdit`) 메인 윈도우 하단에 추가
  2. PubChem `GET /compound/name/{name}/property/IsomericSMILES/JSON` 호출
  3. RDKit `AllChem.Compute2DCoords()` 로 2D 좌표 생성
  4. 좌표를 헥사고날 그리드에 스냅하여 `canvas.atoms`, `canvas.bonds`에 삽입
  5. 애니메이션 없이 즉시 render (`canvas.update()`)

### [계획-B] 분광분석-3D 통합 (입체 구조 레이어 내부 표출)
- **목표**: 스펙트럼 탭을 PDF가 아닌 입체 구조 팝업의 탭으로 표출 (이미 popup_3d.py에 SpectrumPanel 존재)
- **현황**: IR 스펙트럼 탭은 이미 구현됨. ORCA .out 파일 로드 버튼도 있음
- **미구현**: NMR/UV-Vis 탭이 없고, 스펙트럼-분자 연동(피크 클릭 시 해당 결합 하이라이트) 없음
- **다음 작업**: `popup_3d.py`에 NMR 탭 추가 + 피크 클릭 이벤트 연결

### [계획-C] 화학 엔진 DFT 정확도 강화
- **목표**: ORCA/RDKit의 DFT 이론에 맞게 전자구름 계산 로직 수정
- **세부 항목**:
  - 공명 비편재화: Hückel 이론 → π 전자 분포 계산
  - 이온/라디칼: formal charge + 라디칼 전자 개수 반영
  - ORCA 연동: Mulliken 전하를 전자구름 렌더링에 직접 사용
- **작업 파일**: `agents/04_analysis_engine/engine_core.py`, `agents/04_analysis_engine/analyzer.py`
